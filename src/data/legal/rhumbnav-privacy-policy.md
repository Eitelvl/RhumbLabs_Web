## 8. Location Data

The Application requires access to the User's device location to provide core navigation and flight planning services. This section explains how location data is collected, used, and managed.

**8.1 How Location Data is Collected**
Location data is collected through the device's GPS and other positioning technologies when the User grants location permissions to the Application. The Application requires access to location data in the following modes:
- **While using the Application:** Location is accessed only when the Application is actively in use.
- **Precise location:** The Application requires precise location data to provide accurate navigation and flight planning services.

**8.2 How Location Data is Used**
Location data is used exclusively for the following purposes:
- Real-time navigation and map visualization
- Flight planning and route calculation
- Saving flight plan origin and destination points when explicitly entered by the User

**8.3 Storage of Location Data**
RhumbNav operates with a local-first architecture. Data is processed and stored primarily on the User's device.
- Real-time location data used for navigation is processed locally on the User's device and is not transmitted to or stored on the Company's servers.
- Location data is only stored remotely when the User explicitly saves it as part of a flight plan or logbook entry within the Application.
- Remote backup and synchronization through Firebase / Firestore are performed only when the User activates this functionality, is signed into their account, and has access to the corresponding premium features.
- Stored location data is subject to the retention periods described in Section 6 of this Privacy Policy.

**8.4 Sharing of Location Data**
Location data may be shared with Mapbox solely for the purpose of rendering maps and providing navigation services. This sharing is limited to what is necessary for the service. Mapbox's data practices are governed by their own privacy policy, available at https://www.mapbox.com/legal/privacy.

The Company does not share location data with any other third party.

**8.5 User Control Over Location Data**
The User always has full control over location permissions:
- Location permissions can be granted or revoked at any time through the device's operating system settings.
- Revoking location permissions will limit or disable certain features of the Application that depend on location access, including real-time navigation.
- Revoking location permissions does not affect location data previously saved by the User as part of flight plans or logbook entries.

**8.6 Sensitive Nature of Location Data**
The Company recognizes that location data, particularly flight route data, may be sensitive. The Company is committed to processing location data with the highest level of care and limiting its use strictly to the purposes described in this section.
